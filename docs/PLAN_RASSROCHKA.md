| 1,5 мес | 4 | 14 дней | 6,5% | 8,5% |

| 6 мес | 6 | 28 дней | 8% | 10% |

## Шаг 3. Обновление кода — передача параметра рассрочки в платёжную ссылку

### 3.1 Как работает Prodamus Частями технически

Метод активируется через дополнительный параметр в URL платёжной страницы.
По документации Prodamus: `payment_type=bnpl` или аналогичный параметр
(уточнить в кабинете после активации — Prodamus предоставит точный параметр).

### 3.2 Изменения в server.ts → /api/prodamus/pay

Принимать опциональный параметр `installment` (boolean или срок в месяцах).
Если передан — добавить в URL параметр рассрочки.

```typescript
// В теле запроса добавить:
const { tariffName, price, contact, name, installment } = req.body;

// При формировании URL:
if (installment) {
  url.searchParams.append('payment_type', 'bnpl'); // уточнить у Prodamus
}
```

### 3.3 Изменения в src/App.v3.tsx — модалка бронирования

Добавить чекбокс / переключатель «Оплатить в рассрочку» в шаг с тарифами или в форму перед оплатой.

**Рекомендуемое место:** под ценой тарифа или в форме (шаг перед кнопкой «Оплатить»).

**UX-вариант:**
```
[ ] Оплатить в рассрочку через Prodamus Частями
    Первый платёж сейчас, остальное — по графику
```

При выборе передавать `installment: true` в запрос к `/api/prodamus/pay`.
