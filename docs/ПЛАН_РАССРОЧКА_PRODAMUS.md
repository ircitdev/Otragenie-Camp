# План: Подключение рассрочки «Частями» (Prodamus) для тарифов кэмпа

**Сервис:** Prodamus (Rodamus) — платёжная страница `https://dusenko.payform.ru/`  
**Дата составления:** 27 апреля 2026  
**Статус:** Готово к выполнению

---

## Контекст

Prodamus предлагает BNPL-рассрочку «Частями» для платёжных страниц. Тарифы кэмпа «Отражение» стоят от 149 000 ₽ до 249 000 ₽ — рассрочка критически важна для конверсии.

### Условия рассрочки (из ответа поддержки)

| Срок | Кол-во платежей | Интервал | Комиссия (льготный тариф 30 дней) | Стандартная комиссия |
|------|----------------|----------|------------------------------------|----------------------|
| 1,5 мес | 4 | каждые 14 дней | 6,5% | 8,5% |
| 3 мес | 5 | каждые 21 день | 6,5% | 8,5% |
| 6 мес | 6 | каждые 28 дней | 8% | 10% |
| 10 мес | 10 | каждые 28 дней | 12% | 14% |
| 12 мес | 12 | каждые 28 дней | 13% | 15% |

Минимальная сумма рассрочки: 3 500 ₽. Первый платёж списывается сразу при оформлении.

**Рекомендация:** подключить сроки 1,5 мес и 3 мес (наименьшая комиссия). Для кэмпа с оплатой до июня — сроки 6+ мес экономически нецелесообразны: последние платежи придут после мероприятия.

### Этап 3. Технический: UI кнопки рассрочки на странице тарифов

Добавить в карточки тарифов на [src/App.v3.tsx](src/App.v3.tsx) (секция Pricing, ~строка 1390) визуальное обозначение рассрочки.

**Что изменить:**

#### 3.1 Компонент расчёта платежей

Добавить под ценой в каждой карточке тарифа строку:
```
«или от X ₽/платёж — рассрочка на 1,5 мес»
```

Расчёт: `Math.ceil(price / 4)` для 4 платежей (1,5 мес).

Пример для тарифов:
- База 149 000 ₽ → 4 платежа по **37 250 ₽** (1,5 мес)
- Полный 179 000 ₽ → 4 платежа по **44 750 ₽** (1,5 мес)  
- Премиум 249 000 ₽ → 4 платежа по **62 250 ₽** (1,5 мес)

*(Это ориентировочные суммы без учёта комиссии. Точные суммы считает Prodamus на платёжной странице.)*

#### 3.2 Модальное окно бронирования

В модалке бронирования (~строка 2187) — перед кнопкой «Оплатить» добавить чекбокс или информационную плашку:
> «Доступна рассрочка — первый платёж сейчас, остальные автоматически»

#### 3.3 Платёжная ссылка с параметром рассрочки

Изучить API Prodamus: есть ли параметр для включения BNPL при формировании ссылки в `/api/prodamus/pay` ([server.ts](server.ts), строка 124).  
Если параметр есть — добавить его в запрос. Если нет — Prodamus сам показывает метод «Частями» на платёжной странице после подключения (не требует изменений в коде).

**Проверить:** https://help.prodamus.ru/prodamus.-chastyami/kak-oformit-zakaz-s-pomoshyu-servisa-prodamus-chastyami
